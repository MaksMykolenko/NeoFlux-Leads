# Flux Leads — Design System

A design system extracted from the **Flux Leads** product
([`MaksMykolenko/NeoFlux-Leads`](https://github.com/MaksMykolenko/NeoFlux-Leads) — repo
still uses the legacy name but the shipping brand is **Flux Leads**, the
B2B prospecting tool by [NeoFlux Software](https://maksmykolenko.github.io/NeoFlux-Software/)).

Flux Leads is a multi-tenant AI prospecting CRM with three search modes:

1. **Local business** — niche + city → Gemini + Google Search grounding finds
   companies, audits their sites, drafts cold emails.
2. **Beats** — finds music artists who buy type beats (handles, platforms,
   followers, "looking for type beats" signal), attaches a watermarked demo,
   drafts a multi-channel outreach message.
3. **Universal** — free-form prompt → Gemini + Google Search returns any
   kind of lead (events, podcasts, conferences, etc.).

The product is bilingual (uk / en via `next-intl`), **defaults to dark mode**
(Flux ID design tokens), and is auth-gated through Flux ID OAuth with a
Stripe-billed plan ladder (Starter / Pro / Agency).

## Content fundamentals

### Language

The product ships in **Ukrainian and English**. All chrome is translated;
locale lives in the URL (`/uk/...` / `/en/...`).

Code-switching rules (carry into any new copy):
- **Product chrome** in user's locale (uk by default for new Ukrainian users).
- **CRM/sales jargon stays in English**: `Opportunity Score`, `AI Proposal`,
  `Performance Score`, `SSL`, `Mobile-friendly`, `Replied`, `Bounced`,
  `Local` / `Beats` / `Universal` mode names.
- **Status enum values** are English regardless of locale (`New`, `Qualified`,
  `Contacted`, `Replied`, `Won`, `Lost`).
- Cold-email body, generated by Gemini, matches the recipient's likely
  language (uk for Ukrainian businesses, en otherwise).

### Voice & tone

- **Direct, professional, partner-tone**. The Gemini system prompt explicitly
  bans `"Сподіваюсь, у вас все добре"`-style filler and emoji.
- Imperative buttons: `Пошук`, `Зробити аудит`, `Згенерувати листа (AI)`,
  `Зберегти в CRM`, `Копіювати`, `Sign in with Flux ID`.
- Helper text in one sentence where possible; two tops.
- Empty states are short and practical, never apologetic.
- Cold-email copy is 5–7 sentences, no markdown, no `Subject:` prefix, signed
  `"З повагою, команда NeoFlux"` (the agency identity behind Flux Leads).

### Casing

- Page titles, section headings: **sentence case** ("Останні ліди", "Аудит сайту").
- Brand wordmark: `<bold>Flux</bold><bold purple>Leads</bold>` — never break it,
  never put a space between the two words.
- Buttons: **sentence case**, no terminal punctuation.
- Eyebrow labels above cards: `text-xs uppercase tracking-wider`
  (`OPPORTUNITY SCORE`, `AI PROPOSAL`).
- Status pills: **TitleCase** English enum (`New`, `Qualified`).
- Mode badges in the kanban: **ALL-CAPS** `LOCAL` / `BEATS` / `UNIVERSAL`, 9-10px.

### What's banned

- **No emoji** anywhere. The AI system prompt enforces this.
- **No markdown** in user-facing copy.
- **No exclamation marks** in chrome.
- **No corporate filler** ("we strive to", "leverage", "synergize").
- **No long paragraphs** in any UI surface.
- **No accent gradient on UI surfaces** — gradient is reserved for the
  AI button + occasional marketing hero text. Cards stay flat.

### Numbers + dates

- Dates render via `Intl.DateTimeFormat("uk-UA"|"en-US", {…, timeZone: "Europe/Kyiv"})`.
- Scores show as `XX / 100` with `tabular-nums` (Geist Sans handles this natively).
- Follower counts compressed: `4.2K`, `27.8K`, `120K`. The fractional digit
  drops at `≥ 10K`.
- Ukrainian declension applied: `1 проблема` / `2 проблеми` / `7 проблем`.

## Visual foundations

### Dark mode is the default

Set via `next-themes` `attribute="class"` `defaultTheme="dark"`. Every screen
must work in both modes; light is the toggle-away alternative, not the
primary. **Design every new surface in dark first**, then write the light
counterpart with `dark:` siblings on every token-bearing class.

Dark surface anatomy (from `globals.css` Flux tokens):

| Token | Hex | Use |
| --- | --- | --- |
| `--flux-bg` | `#0b0c10` | Page background, body |
| `--flux-card` | `#13141c` | Card surface, header bg, dropdowns |
| `--flux-card-2` | `#1a1b24` | Nested card, hovered card, slightly lifted |
| `--flux-text` | `#ffffff` | Primary text on dark |
| `--flux-muted` | `#8a8d93` | Secondary text on dark |
| `--flux-border` | `rgba(255,255,255,0.05)` | Card border, divider |
| `--flux-border-strong` | `rgba(255,255,255,0.10)` | Hovered border |
| `--flux-purple` | `#6a00ff` | Primary CTA, links, focus ring |
| `--flux-purple-hover` | `#832bff` | CTA hover |
| `--flux-purple-soft` | `#b580ff` | Eyebrow / muted brand text on dark |
| `--flux-purple-tint` | `rgba(106,0,255,0.15)` | Tint background on dark |
| `--flux-purple-ring` | `rgba(106,0,255,0.30)` | Focus ring on dark |

Light surface uses **zinc** neutrals (`zinc-50` page, white cards, `zinc-200`
borders, `zinc-500/700/900` text) and Tailwind `purple-*` for actions.

### Palette

**Primary action** — purple, never blue.
- Light: `purple-600` → `purple-700` on hover; `purple-50` tint; `purple-500` focus.
- Dark: `#6a00ff` → `#832bff` on hover; `purple-tint` background; `#6a00ff` glow on shadow.

**AI accent** — gradient. Used **only** on the "Generate" buttons.
- `bg-gradient-to-r from-violet-600 to-purple-600` (light)
- `bg-gradient-to-r from-violet-600 to-flux-purple` (dark) — same gradient renders fine in both modes.

**Status semantics** (every status pill follows a `-50 / -700 / -200` Tailwind triple in light, and `-500/10 / -300 / -500/30` translucent in dark):

| Status | Color family | Where else it shows up |
| --- | --- | --- |
| New       | **blue** | Kanban accent + funnel segment |
| Qualified | **violet** | Closest in hue to the brand purple — denotes "moving toward Won" |
| Contacted | **amber** | Bulk action bar |
| Replied   | **rose** (NOT purple — purple is the brand) | Kanban + reply-status dropdown |
| Won       | **emerald** | Unlimited-plan UsageMeter, success banners |
| Lost      | **red** | Delete confirmations |

**Score semantics** — Opportunity Score AND Performance Score share the
same thresholds:
- `> 70` → emerald (`text-emerald-600` / `dark:text-emerald-400`)
- `> 40` → amber (`text-amber-600` / `dark:text-amber-400`)
- `≤ 40` → red (`text-red-600` / `dark:text-red-400`)

### Typography

- **Display + body**: Geist Sans `400/500/600/700`.
- Headlines run **bolder than the average product** — h1 is `font-bold` (700),
  the marketing hero is `font-extrabold` (800). Tighten with `tracking-tight`.
- Numerics: `tabular-nums` everywhere they appear (scores, follower counts,
  status counts, plan price).
- **Mono**: Geist Mono — used in admin / settings for SMTP env vars and
  error logs in `<pre>` blocks. Don't sprinkle it through chrome.
- Eyebrow labels: `text-xs uppercase tracking-wider font-semibold`. Color is
  `zinc-500` on light, `zinc-400` on dark, or `flux-purple-soft` (`#b580ff`)
  when it sits inside a marketing section.

### Spacing + layout

- Dashboard container: `max-w-4xl` for the leads list; `max-w-[88rem]`
  (≈1408px) when the Kanban board is active so all six columns fit without
  horizontal scroll on a normal desktop.
- Marketing container: `max-w-7xl` (hero), `max-w-6xl` (pricing, footer),
  `max-w-3xl` (FAQ).
- Card padding: **`p-6`** is canonical for content cards; `p-8` for marketing
  feature cards; `p-3.5` for compact kanban cards.
- Section gap: `mt-6` between dashboard cards; `mt-10` between the action area
  and the table; `py-20` / `py-28` between marketing sections.
- Header height: **`h-16`** (64px), sticky, `backdrop-blur supports-[backdrop-filter]:bg-white/70`.

### Corners + elevation

- **Cards** are `rounded-2xl` (16 px) — bumped up from the earlier `rounded-xl`
  (12 px). Big surfaces (forms, tables, login card) all use `rounded-2xl`.
- **Inner cards / kanban cards**: `rounded-xl` (12 px).
- **Buttons**: `rounded-xl` (12 px) for the big primary CTA, `rounded-lg`
  (8 px) for medium buttons, `rounded-md` (6 px) for small/inline.
- **Pills + badges**: `rounded-full`.
- **Inputs**: `rounded-md`, filled `zinc-50` / `flux-bg`.
- **Marketing hero panels**: `rounded-3xl` (24 px) — only here.

**Shadows**:
- Light: cards use `shadow-sm` (`0 1px 2px rgba(0,0,0,.05)`) — the border
  carries most of the visual separation.
- Dark: cards stack on `--flux-bg` with NO shadow; only the login card and
  modal use `shadow-[0_10px_40px_rgba(0,0,0,0.40)]`.
- **Purple glow** is the signature elevation: `0 4px 20px rgba(106,0,255,.4)`
  on the primary CTA in dark, `0 4px 15px rgba(168,85,247,.4)` (purple-500
  translucent) in light. Three sizes shipped as utilities: `flux-glow`,
  `flux-glow-lg`, `flux-glow-card`.

### Backgrounds

- **Marketing home (dark only)**: `#0a0a0f` with a faint 4-rem CSS grid
  overlay masked by a radial gradient at the top. The hero badge glows
  `#6a00ff` at 1.5 px.
- **Login (dark)**: full-page radial gradient `rgba(106,0,255,0.18) → transparent`
  centered 50% / 30% from the top.
- **Dashboard**: flat `--bg-page`. No textures, no gradients on the page itself.
- **No glassmorphism** except the sticky header (`backdrop-blur`).
- **No imagery** in chrome. User-supplied avatars are the only photographic
  content in the product.

### Animation

- `transition-colors` on hover everywhere — pace `duration-200`.
- `active:scale-95` on every action button. This is the unique press feedback.
- Hover translate: marketing CTAs lift `hover:-translate-y-0.5`; footer links
  shift `hover:translate-x-1` on hover.
- Loading uses `animate-spin` on a 2-quadrant Tailwind spinner.
- `transition-transform` rotates the dropdown chevron 180° on open.
- **No bounces, no springs, no Framer Motion, no Lottie.**
- Kanban drag-and-drop uses `@hello-pangea/dnd` defaults — no custom motion.

### Hover + press states (canonical)

- **Primary button**: `hover:bg-purple-700 hover:-translate-y-0.5 hover:shadow-[0_6px_24px_rgba(106,0,255,0.5)] active:scale-95`.
- **Tertiary / pill**: `hover:bg-purple-100 dark:hover:bg-purple-500/10`.
- **Tables**: row `hover:bg-zinc-50 dark:hover:bg-zinc-800/50`.
- **Links**: `text-purple-600 → text-purple-700 dark:text-purple-400 → dark:text-purple-300`, with `hover:underline`.
- **Cards**: `border-zinc-200 → border-zinc-300 dark:border-flux-border → dark:border-flux-border-strong`.
- **Focus**: `focus-visible:ring-2 focus-visible:ring-purple-500 dark:focus-visible:ring-flux-purple`, always with `ring-offset-2` and `dark:ring-offset-flux-bg`.
- **Disabled**: `disabled:opacity-60 disabled:cursor-not-allowed` (or `cursor-wait` for in-flight). Primary buttons additionally drop to `purple-300` fill in light, `purple-300` translucent in dark.

### Capsules vs. protection gradients

Status, count, and mode classifiers always use **rounded-full capsules** with
1 px inset ring. **No protection gradients** anywhere — the product never
layers text on imagery.

## Iconography

**System**: Heroicons (Tailwind Labs, MIT) — inlined as SVG markup in each
component. No icon font, no sprite, no `<Icon>` wrapper component.

Two flavours:

- **Mini, 20×20, `fill="currentColor"`** — inline UI controls (chevrons in
  selects, clipboard, sparkle, sun/moon, gear, logout). This is the most
  common — most icons in headers, buttons, and pickers are minis.
- **Outline, 24×24, `stroke="currentColor"` strokeWidth=`1.5`** — empty-state
  illustrations (search, document, mail-envelope). Always rendered at
  `w-12 h-12` inside a `rounded-full bg-zinc-100 dark:bg-zinc-800` swatch,
  with `text-zinc-400 dark:text-zinc-500`.

**Custom icons** (drawn inline, also at 20×20 mini-style):
- `BoardIcon` / `TableIcon` — for the view toggle.
- `LockIcon` — for the admin pill.
- `GearIcon` / `LogoutIcon` — header trailing icons.
- `TelegramIcon` / `YouTubeIcon` / `TikTokIcon` — footer social. These are
  brand-traced 24×24 fills.

**Emoji**: never. The Gemini system prompt actively bans them in generated copy.

**Unicode glyphs**: a single `·` (middle dot) is the only separator used in
list rows (`Стоматологія · Черкаси`).

**Logo**: `assets/logo-mark.svg` — solid `#6a00ff` square (`rx=14`) with a
white `N` glyph (`<path d="M19 47V17h6l13 17V17h6v30h-6L25 30v17z"/>`). In
dark mode, the logo always wears a purple glow:
`shadow-[0_0_15px_rgba(106,0,255,0.5)]`. Don't use the gradient or any
alternative fill — the brand committed to flat solid purple.

Wordmark: the brand always reads `Flux` in foreground color, `Leads` in
purple. Footer:
```html
<h3>Flux<span class="text-purple-600 dark:text-flux-purple">Leads</span></h3>
```
